## 2. フロー設計（詳細）

### 2-1. トップレベル分岐

```
入電
 └─ 「本日はどのようなご用件でしょうか？」
     ├─ 予約のキャンセル        → キャンセルフロー
     ├─ 予約の変更/確認         → デフォルトの変更/確認手順
     ├─ 新規予約                 → 新規予約フロー（ステップ1へ）
     └─ その他問い合わせ         → FAQ → 該当なしなら transfer_to_shop
```

### 2-2. 新規予約フロー（厳守・1ターン1質問）

#### ステップ1: 初診の確認
- AI 質問: 「当院の受診は初めてでしょうか？」
- 分岐:
  - 初めて → ステップ1-A
  - 再診 → ステップ2

#### ステップ1-A: 初診の年齢分岐
- AI 質問: 「中学生以上の大人の方でしょうか？」
- 分岐:
  - 中学生以上 → `external_menu_id: 24561`（60分・初診大人）
  - 小学生以下 → `external_menu_id: 24560`（30分・初診小児）
- 次: ステップ3（希望日時の確認）

#### ステップ2: 用件の分岐（再診のみ）
順序は **2-1 → 2-2 → 2-3 → 2-4** の固定。お客様が先に用件を明言された場合は該当分岐へ即ジャンプ。

##### ステップ2-1: 定期検診の確認
- AI 質問: 「本日は定期検診（メンテナンス）のご予約でしょうか？」
- 分岐:
  - 肯定 → `external_menu_id: 25029`（30分・定期検診） → ステップ3
  - 否定 → ステップ2-2

##### ステップ2-2: 詰め物トラブルの確認
- AI 質問: 「詰め物が外れた、または割れたといったお困りごとはございますか？」
- 分岐:
  - 肯定 → ステップ2-2-A
  - 否定 → ステップ2-3

##### ステップ2-2-A: 詰め物の状態確認
- AI 質問: 「詰め物は割れていらっしゃいますか？それとも外れただけでしょうか？」
- 分岐:
  - 割れている → `external_menu_id: 25087`（60分・詰め物割れ）
  - 外れただけ → `external_menu_id: 24603`（30分・詰め物外れ）
- 次: ステップ3

##### ステップ2-3: 来院間隔の確認
- AI 質問: 「前回の当院での治療から半年以上あいておりますでしょうか？」
- 分岐:
  - 肯定・不明 → `external_menu_id: 24562`（30分・半年以上ブランク） → ステップ3
  - 否定（治療継続中）→ ステップ2-4

##### ステップ2-4: 治療継続中の場合
- AI 質問: 「次回の予約内容はすでに決まっておりましたでしょうか？」
- 分岐:
  - 内容が決定済み → オンライン予約案内（既存ロジック維持）
    - SMS 希望 → `send_template_sms` → 通話終了
    - 転送希望 → `transfer_to_shop`
    - LINE/AI受付で取りたい → `external_menu_id: 25088`（その他症状・30分） → ステップ3
  - 内容未定 → `transfer_to_shop`（AI が用件を確定できないため）

#### ステップ3〜6: 既存ロジック維持
- ステップ3: 希望日時の確認
- ステップ4: `get_available_slots`（必ず external_menu_id 指定）
- ステップ5: お名前の確認
- ステップ6: `create_reservation`（必ず external_menu_id 指定）

### 2-3. キャンセルフロー（改善）

```
デフォルトのキャンセル手順:
  search_reservations → cancel_reservation

cancel_reservation 成功直後（新規追加）:
  AI: 「キャンセルを承りました。別日であらためてご予約をお取りいたしましょうか？」
    ├─ 肯定 → ステップ2（用件分岐）から新規予約フローを再開
    │         ※ ステップ1（初診確認）は既知の再診客なのでスキップ
    └─ 否定 → 通話終了案内
```

## 5. リスク・注意

### 5-1. 質問数の増加
ステップ2 で **最大4回**（定期検診→詰め物→半年→治療継続）の確認を行うため、再診の方の通話時間が伸びる可能性がある。

**緩和策**: お客様が「定期検診の予約お願いします」「詰め物が外れて…」など先に用件を明言された場合は、該当分岐へジャンプして他の確認をスキップする旨を絶対禁止事項の直前にメモした。

### 5-2. 担当者制約の見えないずれ
メニューごとに担当者制約があるが（DH 系／院長系で分かれる）、`get_available_slots` がこれを自動考慮する想定。**もし RPA サーバ側で担当者制約が正しく反映されない場合は、Slot が0件返却される事象が増える可能性**がある。

**緩和策**: ビーンズ歯科の予約失敗調査と同じく、dev ブランチの修正群（特に websetting API による担当者制約取り扱い）が本番に当たっているかを併せて確認したほうが安全。

### 5-3. 治療継続中のお客様体験
治療継続中で次回内容が未定の場合は `transfer_to_shop` で受付転送する仕様だが、受付負荷が増える可能性がある。

**緩和策**: 運用後 1〜2 週間の通話ログで「治療継続＋未定 → 転送」の発生頻度を確認し、SMS テンプレ案内などへの切り替えを検討。
