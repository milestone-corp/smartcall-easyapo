## 1. 背景・きっかけ

ゆきデンタルクリニック様より以下のフローで予約を取り分けたいとの要望が来ている：

1. 初めてかどうか → 小学生以下 or 中学生以上に分岐
2. 定期検診かどうか → 定期検診の予約確定
3. ご用件の確認 → 詰め物が割れた場合は60分の予約
3-1. キャンセル希望 → 予約キャンセル＋別日で予約を取るか確認
4. 半年以上空いているか確認 → 再診（30分）の予約確定
5. 内容次第で詰め物が外れた or その他の症状に分岐

一つずつ質問して、最終的に **予約（7種類）or キャンセル or 予約変更** に辿り着くのが理想。

## 2. DOM 調査結果（EasyApo メニューマスタ 7種）

ゆきデンタルクリニック様の EasyApo 管理画面メニューマスタからDOMを取得し、次の7メニューが既に登録済みであることを確認した：

| external_menu_id | 診療メニュー名 | 所要時間 | 主な担当 |
|---|---|---|---|
| **24560** | 当院初めての方（小学生以下） | 30分 | DH1, DH2 |
| **24561** | 当院初めての方（中学生以上） | **60分** | DH1, DH2 |
| **24562** | 前回の治療から**半年以上**あいてしまった方 | 30分 | 院長・ななえDr, DH1, DH2 |
| **25029** | 定期検診（メンテナンス）の予約を希望の方 | 30分 | DH1, DH2 |
| **24603** | 詰め物が**外れて**お困りの方 | 30分 | 院長, ななえDr |
| **25087** | 詰め物が**割れて**お困りの方 | **60分** | 院長, ななえDr |
| **25088** | その他の症状でお困りの方 | 30分 | 院長, ななえDr, DH1 |

### 重要な発見
- お客様要望の「予約（7種類）」と完全一致。**店舗側マスタの追加作業は不要**
- メニュー名表記は「**半年以上**」（既存プロンプトの「5ヶ月以上」と表記がズレている）
- メニュー側に **30分／60分の所要時間が登録済み**のため、AI 側で `duration_min` を指定する必要はない
- メニューごとに担当者制約があり、`get_available_slots` が自動で適切な空き枠を返す

## 3. 現状プロンプト（initial_prompt.md）の不足点

現プロンプトと要望を突き合わせた結果、次の5点が不足していた：

### 不足1: メニュー定義が4種類しかない
現状プロンプトの予約メニュー表に列挙されているのは以下の4種類のみ：
- 24560（初めて・小学生以下）
- 24561（初めて・中学生以上）
- 24562（5ヶ月以上空いた）
- 24603（予定外の症状）

→ **25029（定期検診）、25087（詰め物割れ）、25088（その他症状）が未掲載**

### 不足2: 定期検診の分岐がない
現プロンプトは「初診 → 再診 → 5ヶ月以上空いてる？ → 治療内容決まってる？」のフローのみ。
定期検診（メンテナンス）の選択肢が予約フローに存在しない。

### 不足3: 詰め物が「割れた」と「外れた」が区別されない
既存「その他の追加指示」に自由記述で：
> 詰め物が外れたと言われた場合、割れていないか確認してください。割れている場合は60分の予約、外れただけなら30分の予約を取ってください。

とあるが、これは **`duration_min` を指定する書きぶり**で、`external_menu_id` ベースの厳格運用と矛盾する。25087（割れた・60分）と 24603（外れた・30分）を `external_menu_id` で切り分けるべきところが、プロンプトのフローでは分岐されていない。

### 不足4: キャンセル後の再予約フォローがない
現プロンプトは「予約のキャンセル → デフォルトの手順」のみ。
キャンセル完了直後に「別日で取り直しますか？」と確認するステップが無く、顧客機会を取りこぼしやすい。

### 不足5: 「一つずつ質問」の徹底が弱い
「ステップ1: 初診の確認」で、初診の場合は年齢分岐、再診の場合は5ヶ月分岐＆次回内容確認…と分岐の階層が深く、AI が複数質問を1ターンにまとめがちな構造になっている。お客様要望の「一つずつ質問」が満たしにくい。

### 不足6: 表記揺れ
- プロンプト: 「**5ヶ月**以上あいてしまった方」
- EasyApoマスタ: 「**半年**以上あいてしまった方」

→ EasyApo マスタ（顧客側で運用される表示）に合わせて「半年以上」表記に統一すべき。

### 不足7: 「治療継続中＋予定外症状」の external_menu_id 選択ミス
現プロンプトでは、治療継続中で次回予約内容が決まっており、SMS でも転送でもない選択肢を取った場合は `external_menu_id: 24603`（詰め物が外れた）に流していた。
これは **メニュー名と用途が一致していない**（症状を聞いていないのに「詰め物外れた」枠を予約してしまう）。**25088（その他の症状）に変更**すべき。

## 4. 改善方針（要約）

詳細フローは [20260515-071726_prompt-improvements.md](20260515-071726_prompt-improvements.md) を参照。骨子のみ：

```
ステップ1: 初診の確認（1ターン1質問）
  ├─ 初めて  → 1-A 年齢分岐
  │             ├─ 中学生以上 → 24561 (60分)
  │             └─ 小学生以下 → 24560 (30分)
  └─ 再診   → ステップ2へ

ステップ2: 用件の分岐（1ターン1質問、上から順に確認）
  2-1. 定期検診ですか？     → 25029 (30分)
  2-2. 詰め物トラブル？     → 2-2-A
        ├─ 割れた  → 25087 (60分)
        └─ 外れた  → 24603 (30分)
  2-3. 半年以上空いた？     → 24562 (30分)
  2-4. 治療継続中           → 次回内容確認
        ├─ 決定 → SMS or 転送 or 25088
        └─ 未定 → 転送

ステップ3-6: 希望日 → 空き枠確認 → 名前 → 予約作成

キャンセルフロー: 既存手順 + キャンセル完了後「別日で取り直しますか？」を追加
```

## 6. CS から ゆきデンタルクリニック様に確認すべき事項

導入前に以下を確認してほしい：

1. **質問順序**: ステップ2 で「定期検診 → 詰め物 → 半年 → 治療継続」の順に聞くが、これでよいか？順序を変えると患者が答えやすさを感じる可能性
2. **治療継続中の詰め物トラブル**: 患者が治療継続中で詰め物が割れた場合、25087（詰め物割れ）優先でよいか？ または 25088（その他症状）に流すか？
3. **小学生以下の初診（24560）**: 30分で確実に終わるか？60分枠が必要なケースは無いか
4. **「カルテを作るので10分前にお越しください」案内**: 24560（小学生以下）にも 24561（中学生以上）にも同じく案内するか？
5. **キャンセル後の再予約**: 別日案内時、ユーザーが詰め物トラブル等を口にしたら同じ external_menu_id で取るのか、新たに用件確認するのか
6. **メニュー文言の表記揺れ**: 「半年」 vs 「5ヶ月」のどちらでお客様にご案内するか（EasyApo は「半年以上」表記）
